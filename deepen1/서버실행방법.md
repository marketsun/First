# 🚀 서버 실행 방법

## 가장 쉬운 방법 (추천!)

### Windows 사용자
프로젝트 폴더에서 **`start.bat`** 파일을 더블클릭하세요!

또는 터미널에서:
```bash
start.bat
```

### Mac/Linux 사용자
터미널에서:
```bash
./start.sh
```

---

## 📋 실행 스크립트 종류

프로젝트에는 3가지 실행 스크립트가 있습니다:

### 1. `start.bat` (Windows 전용) ⭐ 추천
- **기능**: 자동 환경 체크 + 패키지 설치 + 서버 실행
- **특징**:
  - Python 설치 확인
  - 필요한 패키지 자동 설치
  - Chrome 브라우저 확인
  - 친절한 안내 메시지
- **사용법**: 파일 더블클릭 또는 `start.bat` 실행

### 2. `start.sh` (Mac/Linux 전용) ⭐ 추천
- **기능**: 자동 환경 체크 + 패키지 설치 + 서버 실행
- **특징**:
  - Python 설치 확인
  - 필요한 패키지 자동 설치
  - Chrome 브라우저 확인
  - 컬러 출력으로 보기 쉬움
- **사용법**: `./start.sh` 실행

### 3. `start_simple.bat` (Windows 간단 버전)
- **기능**: 서버만 바로 실행
- **특징**: 체크 없이 바로 실행
- **사용법**: 파일 더블클릭 또는 `start_simple.bat` 실행

## 📖 단계별 사용법

### 1단계: 실행 스크립트 실행
**Windows:**
- `start.bat` 파일을 더블클릭

**Mac/Linux:**
- 터미널 열기
- 프로젝트 폴더로 이동
- `./start.sh` 입력

### 2단계: 자동 설치 (최초 1회)
- 스크립트가 자동으로 필요한 패키지 설치
- 설치는 최초 1회만 수행됨
- 이후에는 바로 서버 시작

### 3단계: 브라우저 자동 실행 🎉
- **브라우저가 자동으로 열립니다!**
- 만약 열리지 않으면 수동으로 `http://localhost:5000` 접속
- 검색 시작!

### 4단계: 서버 종료
- 터미널 창에서 `Ctrl + C` 키 누르기
- 또는 터미널 창 닫기

## 🔧 수동 실행 방법

자동 스크립트를 사용하지 않고 직접 실행하려면:

### Windows (Git Bash)
```bash
pip install -r requirements.txt
python app.py
```

### Windows (CMD)
```cmd
pip install -r requirements.txt
python app.py
```

### Mac/Linux
```bash
pip3 install -r requirements.txt
python3 app.py
```

## ⚠️ 문제 해결

### 1. "Python을 찾을 수 없습니다"
**원인**: Python이 설치되지 않았거나 PATH에 등록되지 않음

**해결**:
1. Python 설치: https://www.python.org/downloads/
2. 설치 시 "Add Python to PATH" 체크
3. 컴퓨터 재시작

### 2. "패키지 설치 실패"
**원인**: pip 업데이트 필요 또는 권한 문제

**해결**:
```bash
# pip 업그레이드
python -m pip install --upgrade pip

# 패키지 재설치
pip install -r requirements.txt
```

### 3. "포트가 이미 사용 중입니다"
**원인**: 5000번 포트를 다른 프로그램이 사용 중

**해결**:
- `app.py` 파일 마지막 줄 수정:
```python
app.run(debug=True, host='0.0.0.0', port=5001)
```
- 브라우저에서 `http://localhost:5001` 접속

### 4. "Chrome 브라우저를 찾을 수 없습니다"
**원인**: Chrome이 설치되지 않음

**해결**:
1. Chrome 설치: https://www.google.com/chrome/
2. 설치 후 스크립트 재실행

### 5. 스크립트 실행이 안 됨 (Mac/Linux)
**원인**: 실행 권한 없음

**해결**:
```bash
chmod +x start.sh
./start.sh
```

## 💡 팁

### 빠른 재시작
서버가 이미 실행 중이면:
1. `Ctrl + C`로 종료
2. 터미널에서 위쪽 화살표 키
3. Enter (이전 명령어 재실행)

### 백그라운드 실행 (고급)
**Mac/Linux:**
```bash
nohup python3 app.py > server.log 2>&1 &
```

**종료:**
```bash
pkill -f "python3 app.py"
```

### 자동 재시작 (개발 모드)
Flask는 코드 변경 시 자동 재시작됩니다 (debug=True)
